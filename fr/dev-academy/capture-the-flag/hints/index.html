Conseil 3 : Formats#

Transactions, adresses, hauteurs de blocs#

Parmi les indices, vous verrez des chaînes de caractères telles que : 8B74EA4E6EC2B8940E6963DF813FE158171C1FEBEAFA36B38DFE80EA0521970C Il s'agit d'une chaîne hexadécimale de 32 octets. Tendermint/cosmos-sdk hashent les txs et les blocs dans ce format. Le hashage ci-dessus est un hachage de transaction pebblenet. Ces valeurs sont toujours indexées par les explorateurs de blocs. Lancez une recherche avec le hash pour voir son contenu, par exemple : https://block-explorer.pebblenet.cosmwasm.com/transactions/8B74EA4E6EC2B8940E6963DF813FE158171C1FEBEAFA36B38DFE80EA0521970C

Une autre valeur des indices: les chaînes de caractères au format wasm1….. Ces chaînes sont des adresses de compte bech32. Allez-y et examinez-en une avec l'explorateur de blocs: https://block-explorer.pebblenet.cosmwasm.com/account/wasm1k8lesl0wsspcpdfpuh8es3k4ymfsfkryydhw03

La hauteur du bloc est une valeur entière séquentielle non signée qui indexe un bloc. Il peut y avoir de nombreuses transactions dans un seul bloc. En voici un exemple de bloc: https://block-explorer.pebblenet.cosmwasm.com/blocks/334748

Base64#

De Wikipedia : Base64 est un groupe de schémas d'encodage binaire-texte qui représente des données binaires (plus spécifiquement, une séquence d'octets de 8 bits) dans un format de chaîne ASCII en traduisant les données en une représentation radix-64.

Plus simplement, si vous voyez une chaîne de caractères comme celle-ci : aHR0cHM6Ly93d3cueW91dHViZS5jb20vd2F0Y2hcP3ZcPU5VWXZiVDZ2VFBzCg== il s'agit d'une chaîne codée en base64. Pour décoder, il suffit d'exécuter: echo "aHR0cHM6Ly93d3cueW91dHViZS5jb20vd2F0Y2hcP3ZcPU5VWXZiVDZ2VFBzCg==" | base64 -d

Conseil 4 : Requêtes#

Voici comment faire des requêtes sur les données de la chaîne

Les projets basés sur cosmos-sdk/CosmWasm ont tous des apis clients similaires : terracli, wasmd, wasmd et plus encore. Les interfaces de requête et de transaction sont très similaires.

Requête de bloc :

wasmd query block 15 --node https://rpc.pebblenet.cosmwasm.com

Requête de tx :

wasmd query tx 8B74EA4E6EC2B8940E6963DF813FE158171C1FEBEAFA36B38DFE80EA0521970C --node https://rpc.pebblenet.cosmwasm.com

Requête intelligente#

L'interface de requête à un contrat intelligent est définie par le contrat lui-même, ce qui vous permet d'exécuter des requêtes complexes.

Exemple de requête cw20 :

QUERY='{“token_info”:{}}’wasmd query wasm contract-state smart [bech32_address] ‘$QUERY’

Schéma Json#

L'interface d'exécution ou de requête du contrat intelligent est définie par le schéma Json qui se trouve dans le dépôt du contrat intelligent. Le développeur de contrats intelligents doit fournir un schéma aux utilisateurs : https://github.com/CosmWasm/cosmwasm-plus/tree/master/contracts/cw20-base/schema

Exécuter un contrat intelligent#

Vous pouvez exécuter des contrats intelligents à l'aide de cli. Certaines énigmes nécessitent cette compétence pour être résolues :

Envoi de jetons depuis cw20

Vous pouvez envoyer des jetons a partir d'un cw20 avec la commande

SEND_MSG='{“transfer”: {“amount”:”1000″, “recipient”:”wasm170n6mk4k97kvrtj25t9ghm54ewmewt6yq9g6kt”}}’wasmd tx wasm execute $CONTRACT_ADDR “$SEND_MSG” –from wallet

Conseil 5 : Restauration de compte et signature#

Récupération du mnémonique#

La phrase mnémonique est le seul accès à un portefeuille. Vous pouvez récupérer l'accès à un compte si vous avez le mnémonique du portefeuille.

Vous pouvez récupérer un compte en utilisant la mnémonique avec la commande ci-dessous :

wasmd keys add show --recover> Enter your bip39 mnemonicchef sense chicken net around sting course someone question badge hand also nation siren remember famous bird eagle phrase kidney devote damp sugar throw

Signature des transactions hors ligne#

La signature hors ligne est une fonctionnalité intéressante. C'est comme un paiement différé, où vous signez une transaction et ne la soumettez que plus tard au réseau. Un cas d'utilisation : créer une transaction qui enverra du compte A à B et envoyer la tx non signée au propriétaire du compte A pour qu'il approuve et signe la transaction puis la diffuse dans la chaîne.

# –generate-only flag creates tx and only prompts unsigned txwasmd tx bank send wasm134rsdpu5xfhegdclxmdeqxn0j6gmenkmcdmpuz wasm103hx72nfk0mypwlfa3qwyx4rzvv35gvyy83ral 100upebble --memo “you found me” --chain-id pebblenet-1 --generate-only > unsigned_tx.json# {“body”:{“messages”:[{“@type”:”/cosmos.bank.v1beta1.MsgSend”,”from_address”:”wasm134rsdpu5xfhegdclxmdeqxn0j6gmenkmcdmpuz”,”to_address”:”wasm103hx72nfk0mypwlfa3qwyx4rzvv35gvyy83ral”,”amount”:[{“denom”:”upebble”,”amount”:”100″}]}],”memo”:”you found me”,”timeout_height”:”0″,”extension_options”:[],”non_critical_extension_options”:[]},”auth_info”:{“signer_infos”:[],”fee”:{“amount”:[],”gas_limit”:”81363″,”payer”:””,”granter”:””}},”signatures”:[]}
# signs the tx and posts it to the chainwasmd tx sign unsigned_tx.json --from wasm134rsdpu5xfhegdclxmdeqxn0j6gmenkmcdmpuz --chain-id pebblenet-1
